    Key Findings

      Latency Reading Guide

      Sync latency Full HTTP response time for /run. It includes OSCAR Manager, scheduling, execution, output handling, and the returned response.
      Async submit latency HTTP response time for /job. It measures how long OSCAR takes to accept the job, not how long the job takes to finish.
      Async end-to-end Elapsed time from job submission to final state, collected from OSCAR Manager after the load step.
      Async pre-run/run Pre-run time covers the interval from job creation until OSCAR reports execution start. It may include controller reconciliation, admission, scheduling, image handling, and container startup. Run time approximates the service execution lifecycle.
